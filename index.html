Makedown的使用安利以及简单的应用学习指南

半个月前，我在gitpage上搭建了自己的blog。在我想写博文的时候，我发觉在hexo框架下，文章是要用Markdown来排版的。于是我花了点时间来了解Markdown,越发的感觉这种语言对于想写点东西的人来说简直是神器。看到这是不是有点听不懂？其实很简单的，我能学会的你也能学会。

Markdown是什么？我为什么要学它？

Markdown是一种可以使用普通文本编辑器编写的标记语言，通过简单的标记语法，它可以使普通文本内容具有一定的格式。
这是百度百科的定义，是不是很抽象？

我来说说我的理解。相信很多人都用过word排板。手机上麻烦的排版自不用说，在电脑上鼠标点来点去的也很麻烦。学过HTML的同学应该对Markdown会有很熟悉的感觉。Markdown就是用关键字来约定某种格式，来实现只用字符进行排版。实际上，Markdown的输出格式也是HTML格式。不知道什么是HTML的同学也没有关系，并不影响Markdown的使用。

如果你想从事文字行业或者互联网行业，你早晚会接触到Markdown。如果你不想从事这两种行业，你也可以了解一下呀。写一手整洁，可读性高的文字到哪里都是加分项。
让我们看一看Markdown都有哪些优点。

·它是易读（看起来舒服）、易写（语法简单）、易更改纯文本。
·处处体现着极简主义的影子。
·兼容HTML，可以转换为HTML格式发布。
·跨平台使用。
·越来越多的网站支持Markdown。

闲话少叙，让我们看一看如何应用Markdown写作吧！

1.标题

在Markdown编辑器中，在一行的首部输入#，##，###，####这样的字符视为该行的输入为标题。其中，#为一级标题，##为二级标题，以此类推。

举个栗子，输入

# 我是栗子
## 我是栗子
### 我是栗子
#### 我是栗子
##### 我是栗子

会出现以下的效果。

我是栗子

我是栗子

我是栗子

我是栗子

我是栗子

注：Markdown标准写法为文字前加一个空格，如果不加空格有的引擎识别不出来（例如hexo）

2.内容强调与分割线

在输入的文字前后加上 **为加粗，加上*为斜体，加上****为加粗并斜体

继续举个栗子
输入

**我是加粗**

*我是斜体*

****我是加粗加斜体****

会出现以下的效果。
我是加粗

我是斜体

我是加粗加斜体

分割线在一行的开头输入***就可以咯
效果：

3删除效果，高亮及引用

如果你想做出删除某段文字的效果，在某段文字前后加上~~就可以了；如果你想做出高亮的效果，在文字前加 <code> ，在文字后输入</code>就可以了；如果你想做出引用的效果，在一行的开头输入>开始引用，输入两次回车表示终止引用。
接着举栗
输入

~~赫萝一点都不可爱~~
<code>赫萝真可爱</code>
>歪？120吗？这里有个死宅犯病了

会出现以下的效果
赫萝一点都不可爱
赫萝真可爱

歪？120吗？这里有个死宅犯病了

这里需要注意一下，如果引用里需要嵌套结构，与标题类似，>表示第一级，>>表示第二级。

>我是120，请问您的具体地点在哪里？
>>您好，在xx理工大学

我是120，请问您的具体地点在哪里？

您好，在xx理工大学

4.如何输入代码

如果你想输入某段代码，需要在每行代码前输入四个空格
输入

四个空格#include <stdio.h>
四个空格int main() {
四个空格printf(“hello world\n”);
四个空格return 0;
四个空格}

会出现以下的效果

#include <stdio.h> 
int main() {
printf("hello world\n");
return 0;
}

注意，写代码之前需要输入两下回车再开始写代码

5.超链接及图片

如果你想插入超链接，应该按照以下的格式

[超链接文字]（网站链接）
例如我插入我的blog
输入

[sr3的QQ空间](http://Sr3take.github.io)

会出现以下效果：

sr3的QQ空间
如果要超链接图片，与插入网站类似，格式是

![图片文字](图片链接)
blog上还没上传图片，不太好举例子，这里先不举例子了。

6.一些杂项及总结

如果你想输入关键字，而不想让它出现相应的效果，应该在字符前加\;然后需要注意的一点，关键字中所有标点均是英文标点，在写代码时也应注意这件事。

文中写下的只是最基本的Markdown用法。如果还有更多的需求，可以百度，谷歌，上github上寻找相关信息。对于拿什么写Markdown，我是Notepad++一把梭。还有一个网站也不错。网址是：http://md.aclickall.com/
本文大部分都是拿手机在这个网站上写的，然后再用Notepad++修改细节。

在可预料的未来，我感觉Markdown潜力无限。我会一直分享一些我认为非常有用的东西给大家，希望都能学到更多的东西。
